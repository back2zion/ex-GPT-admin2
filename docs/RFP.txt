
FUN-001
생성형AI 시스템 개선
 기 구축된 생성형 AI 시스템 기능 개선
 o 운영중인 LLM 모델보다 향상된 최신 LLM 모델 구축
  - 기본모델은 현재 운영중인 70B 모델보다 상위 모델 적용
    * 최신 LLM 모델은 전/후 성능검증을 통해 적용(FUN-006 참고)
  - 멀티모달 LLM 적용을 통한 이미지 검색 기능 개발
    * 멀티모달 LLM은 2개 이상 구축해 신뢰도·안정성 검증 후 선정
  - 외부망 연결 없이 구축형 서비스 지원
  - 기본모델과 멀티모달은 통합된 화면에서 동시 운영
 o 번역, 요약, 보도자료, 나만의 AI 등 기능 개선
  - 입력된 언어(한국어, 영어 등)을 자동인식하여 번역결과 제공
  - 나만의 AI는 2개 이상 문서 업로드를 통한 비교분석 기능 추가
  - 첨부파일은 100Mbyte 이상 업로드 가능
 o 우리공사 모바일오피스와 연계한 모바일 환경 구축
  - 음성↔문자 변환·인식 기능을 활용한 회의록 자동 요약·정리 기능 개발
  - 요약·정리된 회의록 사내 메일 송출
 o 기존 레거시 시스템과 연계한 업무 지원 기능 개발
  - 전자조달 등 레거시 시스템과 연계
  - 인력정보 등 DB에 적재된 데이터 연계
    * 퇴직추계액, 호봉, 수당 등 개인별 조회
 o 부서별로 학습데이터 참조 범위를 지정할 수 있도록 구성
  - (예) 국가계약법 → 전부서 참조, 야생동물보호법 → 품질환경처만 참조
 o Web/Was 이중화 구성
 o 생성형 AI 모델을 학습시킬 수 있는 환경 개선
  - 학습데이터 관리 기능 개선
  - 사전 학습 모델 추가 학습 (Fine tuning) 기능 개선
  - RAG(Retrival-Augmented Generation) 기반 검색 환경 개선
 o 운영중인 서비스 UI/UX 등 사용자 환경 개선



 FUN-002
 생성형AI 활용 관리도구 기능개선
 학습데이터 및 대화이력 관리               
 o 레거시 시스템과 DB 연계를 통한 제·개정 문서 확인 기능 개발
  - 법령, 사규, 업무기준 등 변동 시 관리자 확인 기능 추가
  - 제·개정된 문서는 변동된 부분만 전처리에 자동 반영
 o 서비스 관리 기능 개선
  - 서비스 평가 및 개선을 위한 사용 이력 관리 (질문/답변/시각 등)
  - 접근 가능 문서 권한 관리 (부서별, 결제라인별 등)
  - 이용만족도 조사 기능
  - 공지메시지 표출 기능



 FUN-003
 생성형AI 활용 서비스를 위한 학습데이터 갱신 및 추가
 우리 공사 내부 자료를 생성형AI 서비스에 적용하기 위한 전처리
 o 생성형AI 활용 서비스를 위한 학습데이터 갱신
  - 법령, 사규, 규정, 매뉴얼, 지침, 각종 보고서 등 우리 공사 내부 자료 중 기존에 학습된 데이터 최신화
  - R&D보고서, 내부방침, 보도자료, 알리오 공시자료 등 학습데이터 추가
  - 발주처 관련 부처를 통한 데이터 수집
  - 데이터 유형별 분석을 통한 전처리 자동화 방안 제시
  - 데이터 정제·변환·분할 등 전처리를 통한 학습데이터 구축
 o 감사, 안전, 재난, 기술심사 등 특정업무 맞춤형 데이터 구축
  - 과거사례, 징계양정요구기준 등을 참조하여 처분수준 제안
  - 감사 유사사례별 감사의견 검색 및 제안
  -  공종별 다양한 유형의 위험성평가결과를 취합하여 하나의 표준 위험성평가 도출
  - 재난상황별 대응 매뉴얼 학습을 통한 신속한 재난 대응 지원
  - 기술 규격, 평가기준 등을 학습하여 심사업무 지원
 o 민원답변 음성데이터 학습데이터 변환
  - 고객민원 응대를 통해 저장된 음성 데이터의 문자 변환(STT)
  - 문자로 변화된 민원 데이터는 학습데이터로 정제
 o 전처리 데이터 개인정보 유무 검출
  - 개인정보 의심 데이터가 포함된 원본 문서 목록
  - 검출된 데이터의 관리자 확인



 FUN-004
 대규모 언어 모델 (LLM)에 학습데이터 반영
 구축된 학습데이터를 LLM에 반영하여 우리 공사용 LLM 구축
 o 생성형AI 활용 서비스 및 학습데이터 유형에 따른 튜닝 방법 선정
   * 예시) 검색 서비스를 위한 RAG 기법, 문서 생성을 위한 Fine Tuning
 o 구축된 학습데이터와 튜닝 방법에 따라 LLM 튜닝 
  - 필요 시, 용도에 따라 다르게 튜닝된 다수의 LLM 구축
  - 영어기반 모델 기반이더라도 한국어로 원활한 소통이 되어야 함
  - 대화의 맥락이 이어지는 Multi turn 형태의 대화
  - 튜닝 결과에 따라 편향성 제거 및 재학습을 통한 모델 품질 유지
 o 우리공사에서 별도의 데이터 전처리 프로그램 도입 시 연계하여 활용




 FUN-005
 생성형AI 활용 서비스 개선
 우리 공사용 LLM을 활용한 사내 메신저 챗봇 서비스 개발
 o 지능형 검색 기능 개선
  - 사용자의 문장 형태 질의 맥락을 파악하여 적절한 자료 요약 제공
  - 답변 근거 문서 링크 및 다운로드 제공
  - 검색에 활용되는 문서는 시스템 연계를 통해 자동으로 최신화
    * 연계할 문서 시스템은 발주처와의 협의를 통해 결정
 o 문서 사전 검토 기능 개선
  - 사용자가 업로드하는 문서의 내용을 관련 규정에 비추어 검토
  - 보완이 필요한 부분을 발췌하여 답변
  - 다양한 업로드 문서 포맷 지원 (txt, pdf, hwp, hwpx 등)
 o 문서 생성 기능 개선
  - 문서 요약 기능(1줄 요약, 2줄 요약 등 추가)
  - 문서 번역 언어 추가(중국어/일본어 등)
     * 추가 언어는 발주처와의 협의를 통해 결정
  - 보고서 1페이지 요약
