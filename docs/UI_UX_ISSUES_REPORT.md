```
         현재 위치
```

### 주요 문제점

#### 1. Backend API 및 Data 처리 기능 개발 부재

- **현상**: Backend API 및 Data가 Mock 상태
- **영향**: 계획대로 구동되는 프로그램 상황이 아니므로 주어진 기능을 테스트할 수 없음
- **필요 조치**: 개발이 기획/설계안대로 개발 완료한 후에 테스트를 진행할 수 있음

**Mock 상태인 주요 기능들:**
- 대상 문서 관리 (접속 불가)
- 사전 관리 (접속 불가)
- 오류신고 관리 (접속 불가)
- 추천질문 관리 (접속 불가)
- 버전 관리 (API 없음)
- 모델 배포 Health Check (TODO 상태)

#### 2. 기능 정의 부재

- **현상**: 각 메뉴별 기능이 동작 가능한 메뉴 기준으로 CRUD를 수행한다고 하나, Backend API 및 Data가 Mock 상태
- **영향**: 잘 구동되는 프로그램 상태가 아니므로 기능이 완성되었다고 보기 어려움
- **결과**: 테스트를 진행할 수 없음

### 다음 기능 테스트 시 고려사항

테스트 목록 기반으로 테스트 담당자가 테스트를 진행할 수 있도록 다음 사항이 필요합니다:

1. **각 기능별 정의**
   - 메뉴별 CRUD 상세 명세
   - 비즈니스 로직 정의
   - 데이터 모델 정의

2. **각 기능별 정의를 적용한 프론트엔드/백엔드 개발**
   - 실제 API 엔드포인트 구현
   - 실제 데이터베이스 연동
   - Mock 데이터 제거

3. **각 기능별 기능 테스트 목록**
   - 단위 테스트 케이스
   - 통합 테스트 시나리오
   - E2E 테스트 시나리오
